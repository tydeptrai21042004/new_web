# Financial Fraud ONNX Web Demo — Single Vercel Version

This is the corrected **single Vercel deployment** version of the financial-statement fraud detection demo.

The previous version used:

```text
Next.js frontend + FastAPI backend
```

This version uses:

```text
Next.js frontend + browser-side ONNX Runtime Web + small Next.js health route
```

That means you can deploy everything as **one Vercel project**.

---

## Why this version is different

A normal Vercel Function is not suitable for uploading a MiniLM ONNX model because ONNX files are usually much larger than the request-body limit for Vercel Functions. Therefore, this app does **not** upload the ONNX model to a server.

Instead:

```text
User browser
→ uploads web_model_bundle.zip locally
→ loads model.onnx with onnxruntime-web
→ tokenizes text using vocab.txt
→ runs MiniLM ONNX inference
→ applies classifier_weights.json
→ shows fraud probability
```

No Python backend is required.

---

## Required model files

Recommended upload file:

```text
web_model_bundle.zip
```

It should contain:

```text
model.onnx
vocab.txt
classifier_weights.json
model_metadata.json optional
```

This bundle is generated by the corrected Kaggle/Jupyter notebook.

You can also upload files separately:

```text
model.onnx
vocab.txt
classifier_weights.json
```

## Run locally

```bash
npm install
npm run dev
```

Open:

```text
http://localhost:3000
```

Then upload `web_model_bundle.zip` in the web UI.

---

## Deploy to Vercel

### Option 1: GitHub import

1. Push this folder to GitHub.
2. Go to Vercel.
3. Import the GitHub repository.
4. Keep the default Next.js settings.
5. Deploy.

### Option 2: Vercel CLI

```bash
npm install -g vercel
vercel login
vercel
```

---

## Important limitation

The model is loaded in the browser memory. If the page is refreshed, upload the model bundle again.

This is intentional so the app can deploy cleanly on a single Vercel project without external storage.

---

## API route included

The app includes one lightweight route:

```text
GET /api/health
```

It only confirms that the Vercel app is running. Prediction is performed in the browser.
